# Buttons

To identify quickly the different actions the user can do inside a module, we create different type of buttons. We can distinguish the **Big-buttons** and the **Small-buttons.** For each of them, we have 3 kind of priority: *secondary-action*, *main-action*, *top-action*.

### Top action

**"Top action"** buttons are only used next to **"main action"** button when the priority is higher: e.g.: **"save & publish"** is an highest action than **"save"** because you can't roll back after publishing.

## Secondary button

### States & Usages

Secondary buttons are mainly used to **"second or third level"** actions or to **"roll back" "cancelled" "deleted"...**

### Don't

Don't use a Big button inside tables.
